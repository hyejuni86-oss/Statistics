## 01. 회귀모형의 형태 바꾸기

```
✅ 학습 목표 :
* 이원배치 분산분석에 대해 설명할 수 있다.
* 비선형회귀에 대해 설명할 수 있다.
```
상호작용 <br>
: 설명변수들끼리 서로 영향을 주면서 y에 대한 효과가 달라지는 것 <br>
현실 데이터에서는 어떤 설명변수가 증가할 때 y가 변하는 방식이, 다른 설명변수 값에 따라 달라질 수 있음<br>
#이런 관계를 회귀모형에서는 보통 변수끼리 곱한 형태로 넣어서 표현함<br>

이원배치 분산분석 <br>
: 요인이 2개인 분산분석 <br>

## y = β0 + β1X1 + β2X2 + γ12X1X2 + ε <br> 

주효과: 다른 요인을 따로 보지 않고, 각 요인 하나만 봤을 때의 효과 <br>

상호작용: 두 요인이 함께 작용해서 나타나는 추가 효과<br>

가설검정 결과 상호작용항이 유의하지 않으면 상호작용은 없다고 보고, 각 주효과를 중심으로 해석함
<br>
비선형회귀 : x에 대해 비선형 형태인 모형을 적용하는 회귀분석<br>
무조건 복잡한 모형을 쓰는 게 좋은 건 아님<br>
해석이 중요한지, 예측이 중요한지, 과적합 위험이 있는지 등을 같이 보고 모형 복잡도를 결정해야 함 <br>



## 02. 일반화선형모형의 개념

```
✅ 학습 목표 :
* 선형회귀가 적절하지 않은 상황을 판단할 수 있다.
* 가능도와 최대가능도 방법을 이해한다.
* 로지스틱 회귀와 오즈비에 대해 이해한다.
* 푸아송 분포와 푸아송 회귀에 대해 이해한다.
* 다양한 일반화선형모형에 대해 설명할 수 있다.
```
일반선형모형: 설명변수가 양적 변수인 다중회귀부터, 범주형 변수인 분산분석까지 포함하는 넓은 개념의 모형<br>
y = a + b1x1 + b2x2 + ... + bkxk + ε <br>
<br>
일반화선형모형(GLM) <br>
: 일반선형모형을 확장한 것으로, 최소제곱법이 아니라 확률분포를 바탕으로 한 최대가능도방법으로 모형을 추정함 / 반응변수가 꼭 연속형일 필요 없이, 더 다양한 형태의 데이터 분석 가능
<br>
통계모형화: 데이터의 성질을 고려해 확률모형을 세우고, 파라미터를 추정하고, 모형을 평가하는 일련의 과정
<br>
일반선형회귀가 적절하지 않은 경우
- 반응변수가 0/1처럼 두 가지 값만 가질 때
- 반응변수가 음이 아닌 정수일 때
- 데이터 퍼짐이 일정하지 않아 직선으로 설명하기 어려울 때
<br>
최대가능도방법 : 데이터와 모형 사이 거리를 최소화하는 방식이 아니라,
주어진 파라미터에서 현재 데이터가 얼마나 그럴듯하게 나올 수 있는지를 기준으로 모형을 평가하는 방법<br>
<br>
가능도(우도)<br>
P(x|θ) = P(x1|θ) × P(x2|θ) × ... × P(xn|θ)<br>
<br>
θ : 확률분포의 파라미터<br>
가능도가 크다는 것은, 해당 θ에서 지금 관측한 데이터가 나올 가능성이 높다는 뜻<br>
최대가능도추정: 가능도를 가장 크게 만드는 θ를 찾아 추정값으로 사용하는 방법<br>
보통은 계산 편의를 위해 로그를 취한 로그가능도를 사용함
<br>
로지스틱 회귀 : 반응변수가 두 가지 범주를 가질 때 사용하는 회귀모형
-한 범주가 일어날 확률을 p라고 두고,설명변수 x가 변할 때 이 확률 p가 어떻게 달라지는지 분석함 <br>
-yi ~ B(N, pi)<br>
-pi = 1 / (1 + exp(-(a + bxi)))<br>
<br>
로지스틱 함수: a + bx 값을 0~1 사이 확률값으로 바꿔주는 함수<br>
-실제 관측값은 0 또는 1이지만,회귀식에서는 그 사건이 일어날 확률 p를 모델링함<br>
- 정의역 : (-∞, +∞)<br>
- 치역 : [0,1]<br>
- 이항분포의 확률을 표현하기에 적절하고, 수학적으로 다루기 편함<br>
a : 그래프의 위치를 좌우로 이동시키는 역할<br>
b : 변화의 크기와 방향을 결정하는 역할<br>
<br>
오즈비 /<br>
오즈(승산): 어떤 사건이 일어날 확률 / 일어나지 않을 확률<br>
즉, 확률 p를 0~∞ 범위의 값으로 바꿔서 본 것<br>
<br>
오즈비: 두 확률 p, q의 오즈를 비교한 비율
<br>
-OR = [p / (1-p)] / [q / (1-q)]<br>
-OR > 1 : p 쪽 사건이 더 일어나기 쉬움<br>
-OR < 1 : q 쪽 사건이 더 일어나기 쉬움<br>
<br>
푸아송 분포: 어떤 사건이 평균적으로 λ번 일어날 때, 실제로 몇 번 일어나는지를 설명하는 확률분포<br>
- 파라미터는 λ 하나만 있음
- λ는 평균이면서 동시에 분산임
-평균과 분산이 서로 독립이 아님

## 04. 통계 모형의 평가와 비교

```
✅ 학습 목표 :
* 왈드 검정과 가능도비 검정에 대해 설명할 수 있다.
* AIC와 BIC에 대해 설명할 수 있다.
```

<br>
왈드검정 : 최대가능도추정으로 얻은 추정값과 표준오차를 이용해 검정하는 방법<br>
-왈드통계량= 추정값 / 표준오차<br>
-최대가능도추정량이 정규분포를 따른다고 보면,왈드통계량으로 신뢰구간이나 p값 계산 가능<br>
<br>
가능도비검정: 두 모형을 비교해서, 더 복잡한 모형이 데이터 적합도를 유의하게 개선했는지 보는 방법 <br>
- 비교하는 두 모형은 반드시 내포관계여야 함<br>
- 부트스트랩 방법: 데이터를 반복적으로 다시 만들어 추정량의 성질을 살펴보는 방법
<br>
AIC: 새로운 데이터를 얼마나 잘 예측할 수 있는지를 기준으로 모형을 평가하는 지표<br>
-AIC = -2logL* + 2k<br>
-L* : 최대가능도<br>
-k : 파라미터 개수<br>
-AIC가 가장 작은 모형을 선택<br>
-과대적합: 현재 데이터에만 과하게 맞아서 새로운 데이터에는 잘 맞지 않는 상태<br>
<br>
BIC
-BIC = -2logL* + klog(n)<br>
-L* : 최대가능도<br>
-k : 파라미터 개수<br>
-n : 표본크기<br>
-값이 가장 작은 모형을 선택
<br>


## 06. 가설검정의 문제점

```
✅ 학습 목표 :
* 왜 유의수준 α = 0.05를 사용하는지 설명할 수 있다.
* 피셔류 검정과 네이만-피어슨류 검정 체계의 이론적 차이와 적용 목적을 설명할 수 있다.
* 효과크기를 이해한다.
* 베이즈 인수를 이해한다.
```
<br>
P-value: 귀무가설이 참이라고 가정했을 때, 지금 관측한 결과보다 같거나 더 극단적인 결과가 나올 확률<br>
-p값이 작다: 귀무가설과 실제 관측 데이터 사이 괴리가 크다는 뜻 <br>
-유의수준보다 작으면 귀무가설을 기각함<br>
- α=0.05쓰는 이유: 유의수준 α와 제2종오류 확률 β는 서로 반대 방향 관계라서,
α를 너무 작게 만들면 β가 커져서 검정력이 떨어질 수 있음<br>
<br>
피셔류 검정: p값을 계산해서 귀무가설과 데이터 사이 괴리 정도를 평가하는 방식<br>
- 가설 기각/채택을 딱 잘라 말하는 데 초점이 아님<br>
- p값이 작을수록 귀무가설과 안 맞는다고 해석<br>
<br>
네이만-피어슨류 검정: p값이 α보다 작은지만 보고 기각/채택을 결정하는 방식<br>
- 검정 전에 α, β, 효과크기, 표본크기를 미리 설계함<br>
- p값이 얼마나 작은지는 크게 중요하지 않고, 기준 넘었는지가 중요<br>
<br>
표본크기 n의 결정: 네이만-피어슨 검정에서는 데이터를 보기 전에 미리 결정해야 하는 요소<br>
-α, 1-β, 효과크기, 표본크기 n<br>
-이 네 가지 중 세 개를 정하면 나머지 하나가 결정됨<br>
-표본크기가 커질수록 p값은 작아지기 쉬움-->표본이 많으면 아주 작은 차이도 유의하게 나올 수 있음
<br>

베이즈 인수 : 두 모형 중 어떤 모형이 데이터를 더 잘 설명하는지 비교하는 지표<br>
-주변가능도<br>
-p(x|M) = ∫ p(x|θ,M)p(θ|M)dθ<br>
-M : 모형
-θ : 모형의 파라미터
-어떤 모형이 데이터 x를 얼마나 잘 설명하는지 나타내는 값
-베이즈 인수
-B = p(x|M1) / p(x|M2)
-B > 1 : M1이 데이터를 더 잘 설명함
-B < 1 : M2가 데이터를 더 잘 설명함
<br>
좋은 가설과 FDR<br>
허위발견율 FDR: 옳다고 판단한 결과들 중 실제로는 틀린 결과의 비율<br>
좋은 가설: 실제로 참일 가능성이 높은 가설, 즉 진실을 더 잘 반영하는 가설<br>



## 07. p-해킹

```
✅ 학습 목표 :
* p-해킹의 개념과 특징을 설명할 수 있다.
```

p-해킹: 의도적이든 아니든 p값이 0.05 미만이 되도록 분석과 해석을 조정하는 행위 <br>
-숫자를 직접 고치는 건 아니지만 유의한 결과가 나올 때까지 실험이나 분석 방향을 바꾸는 것<br>
-p<0.05가 될 때까지 표본크기를 계속 늘림<br>
-여러 설명변수 중 유의한 것만 골라 보고함<br>
-여러 분석 방법을 시도한 뒤 유리한 결과만 채택함<br>
<br>
HARKing: Hypothesis After the Results are Known 결과를 먼저 보고 난 뒤 가설을 만드는 것
<br>
-의미 없는 데이터를 많이 보다 보면 우연히 의미 있어 보이는 결과가 하나쯤 나올 수 있어서 재현성을 떨어뜨리는 대표 원인 중 하나임.<br>

  연구자는 최종 결과(p = 0.03)만을 논문에 보고하며  
  “신약은 통계적으로 유의한 효과가 있다”고 결론 내렸다.**

> **다음 중 위 상황을 가장 적절하게 설명한 것은 무엇인지 고르고 선택한 답의 이유를 간단히 서술하시오.**

~~~
1️⃣ 연구자는 데이터를 정제했으므로 통계적으로 더 정확한 결과를 얻은 것이다.  
2️⃣ 분석을 반복했으므로 제2종 오류가 감소한 것이다.  
3️⃣ 유의한 결과만 보고했으므로 p-해킹 및 제1종 오류 증가의 가능성이 있다.  
4️⃣ p값이 0.05보다 작으므로 과학적으로 효과가 증명되었다.
~~~
